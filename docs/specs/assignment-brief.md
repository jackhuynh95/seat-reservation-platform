# Assignment Brief

## Assessment

Build a small public seat reservation platform using any TypeScript stack.

The assessment evaluates:

- engineering judgment;
- system design thinking;
- software quality;
- operational awareness;
- security considerations;
- thoughtful trade-offs;
- handling of real-world failure cases.

The evaluator allows AI-assisted development, but the assessment focuses on technical decisions, architecture, operational thinking, edge cases, and explainability.

## Scenario

The application should display 3 available seats that can be reserved by authenticated users.

Users should be able to:

- log in with session expiry at 90 days;
- select a seat;
- proceed to payment;
- successfully reserve the seat when payment completes.

## Candidate Freedom

The assignment allows choosing:

- framework;
- database;
- infrastructure approach;
- authentication provider;
- mock payment provider;
- deployment method.

## Submission

Submit a zip file containing:

- source code;
- README documentation;
- setup instructions;
- environment configuration guidance;
- scripts or documentation required to run locally.

## Time Expectation

The assignment is expected to take approximately 2 hours of focused implementation time.

## Important Review Context

The provided review checklist is stricter than the assignment brief. It treats microservices and a message broker as mandatory. This repository should therefore be planned as a microservices submission from the start.
