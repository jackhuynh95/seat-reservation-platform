Payment flow:

+---------+        +-----------------+
| Gateway |------->| Payment Service |
|  :3000  |        |      :3003      |
+---------+        |-----------------|
                   | Create intent   |
                   | Mock pay        |
                   | Verify webhook  |
                   | Publish event   |
                   +--------+--------+
                            |
                            | payment.completed.v1
                            | payment.failed.v1
                            v
                   +-----------------+
                   |    RabbitMQ     |
                   |      :5672      |
                   +--------+--------+
                            |
                            | consume
                            v
                   +-----------------+
                   | Payment Worker  |
                   |      :3004      |
                   +--------+--------+
                            |
                            | internal apply event
                            v
                   +-----------------+
                   | Seat Service    |
                   | Reserve/release |
                   +-----------------+

Rule:
- User actions go through Gateway.
- Payment result goes through RabbitMQ.
- Seat Service owns final seat state.
